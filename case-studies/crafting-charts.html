Crafting charts

qunb - 2015

At qunb, our mission is to make data easy to understand. To do so, we created a document called a "datastory" : series of slides that deliver simple messages illustrated by charts. We really tried to keep things simple : one slide, one message.

When we decided to redesign our charts, one of the first decisions was to always show the data that we're talking about. Before that, users had to explore the chart to discover the numbers related to the message. Exploration is not the first use case of our charts. The message is clearly the first use case, so it was important to put everything in that direction.

Colors

Managing colors has been challenging since we use one color for each "scenario" of datastory.
For each color, we had to create a palette of 7 shades. These 7 shades will be used in 2 cases :

  • Sometimes we need to convey information through the color (different shades = different ranges of values as for a heatmap for instance).
  • And sometimes we just need to use different shades in order to improve the readability of the chart.

Charts types

A large part of the work consisted in identifying charts types from charts options. I think that this is sometimes a completely subjective question. For instance, is a donut chart an option of a classical pie chart ? Or are they completely different charts ?

Animations

With the datastory, we created a new kind of document. I guess that trying to mimic existing solutions when innovating is a common behavior, and we often display charts on a screen in the same way that they were printed on paper.

One major improvements that our new medium can bring, comes from animations.
We are now able to convey information in the way we animate our charts.

For instance, the marimekko chart has always been one of the most difficult to understand. Beyond the difficulty to compare areas, the problem is that there is a lot of dimensions (a measure, some categories, some sub categories for each category,...), and the relationship between each of them is not clear.

By animating the way we build the chart, we can help to explain how it should be read.

We start by showing main categories from the left to the right, and then we draw sub categories inside each category from the bottom to the top.
